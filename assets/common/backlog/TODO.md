# TODO Backlog

## Defines
- Unscheduled task inbox.
- 明確な計画を立てる前に発生した「やるべきこと」の一次置き場。
- 流動的に追加・統合・分割・撤回できる運用対象。

## Inbox
- [ ] `<short-title>`
  - Context:
  - Why now / Why later:
  - Next check timing:
  - Related docs/issues:

## Promotion rule
- 「今やる」と決めた項目のみ Issue 化する。
- Issue 化した項目は `backlog/INDEX.md` と `backlog/issues/` に反映する。
- Issue 化後は TODO から削除する。
