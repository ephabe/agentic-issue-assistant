## Defines
- UX behavior contract.
- 画面ごとの状態遷移（loading/ok/error/empty）を定義する。
- 更新・再取得ポリシーを定義する。

## Routes
### `<route>`
- Purpose: （この画面の目的）
- Primary actions: （ユーザーがこの画面で行う主要操作）
- Required data: （表示に必須のデータ）
- Access: （認証/認可の条件）
- Notes: （表示条件や遷移の補足）

## Global states
### loading
- Trigger: （いつ loading に入るか）
- UI behavior: （スケルトン/プログレス/操作可否）
- Timeout policy: （何秒で別状態へ遷移するか）

### ok
- Success condition: （何を満たせば ok とみなすか）
- Must show: （ok時に必ず見える要素）

### error
- Trigger: （どの失敗で error にするか）
- Message policy: （ユーザーに何を伝えるか）
- Recovery action: （retry/戻る/問い合わせ導線）

### empty（必要な場合）
- Trigger: （0件などの条件）
- UI behavior: （空状態の説明と次アクション）

## Refresh policy
- UI refresh interval: （自動更新間隔。不要なら `none`）
- Manual refresh triggers: （再読み込みボタン、再訪時など）
- Auto refetch triggers: （画面遷移復帰、更新操作完了後など）
- Cache invalidation: （作成/更新/削除後の無効化条件）
- API cache: （Cache戦略の有無、TTLなど）
- Consistency target: （いつまでに最新状態へ反映されるべきか）
